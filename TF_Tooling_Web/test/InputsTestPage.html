Screen Control Client for Web Testbed

To test the performance of the pointerenter / pointerleave event transmission in ScreenControl Client for Web, please do the following in a Chrome browser:

  1. Ensure that the Screen Control Service is running
  2. Ensure that this page is the active client of ScreenControl service.
    1. To do this, simply refresh the page. This will reconnect the page to the ScreenControl Service.
    2. Make sure not to open another ScreenControl Client during the test, such as the Service's UI or another web page that uses ScreenControl Client.
  3. Check the expected visual behaviour by moving the mouse from top to bottom of screen and vice versa. The visuals when using the Screen Control Cursor should mirror mouse behaviour.
  4. Open the developer tools (F12), then open the command window (ctrl+shift+p) and type "show frames". Click on the "Show frames per second (FPS) meter"
  5. Now start moving the cursor around the green and yellow areas. Keep note of both the visuals and the percentage in the FPS meter.
    The visuals should update smoothly between each sub-region of the green and yellow areas, as well as when switching between green and yellow areas.
    Pay particular attention to the behaviour when switching between the green and yellow areas as these are the transitions at which the most events will be sent.
    The percentage should not drop below 95%.
